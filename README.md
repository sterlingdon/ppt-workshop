# PPT Workshop

PPT Workshop is not an article summarizer, a template filler, or a one-click slide generator.

It is a presentation-making workflow where agents act as the editorial, design, production, and review team behind a visually striking, high-fidelity deck.

## What We Believe

A good PPT is not a compressed article. It is a decision artifact.

The job of a deck is to help a specific audience understand what matters, remember the core argument, and make better judgments. That means the workflow starts with audience, thesis, emphasis, and omission before it ever becomes a visual artifact.

We do not follow the source material mechanically. We reshape it into a presentation narrative.

## The Agent's Role

In this project, the agent is not a passive operator of tools.

The agent is responsible for judgment:

- deciding what the deck should say
- identifying who the deck is really for
- cutting weak or irrelevant material
- turning the approved content into a visual direction
- creating slides with hierarchy, rhythm, and intent
- inspecting the rendered result like a visual director
- repairing anything that is technically valid but not presentation-grade

The tools support the workflow. The agent owns the quality.

## What We Produce

We produce high-fidelity PowerPoint decks for real presentation, reporting, research, strategy, and decision-making contexts.

These decks are expected to be:

- audience-first, not article-first
- visually coherent, not a collection of unrelated pages
- designed around one clear idea per slide
- readable at presentation distance
- grounded in the source, without invented facts
- visually striking enough to feel designed, not merely generated
- polished enough to be reviewed visually, not only validated technically

The final deck should feel authored, not assembled.

The PPTX is not just an editable echo of HTML. HTML is the rendering and review surface; the PowerPoint file is a first-class final deliverable. It should preserve the visual ambition of the browser rendering, even when that requires deliberate export tradeoffs.

## The Quality Bar

Passing a technical check is necessary, but it is not enough.

A slide can contain all required text and still fail because it has no focal point, weak hierarchy, poor spacing, generic layout, or no real value for the audience.

Visual craft is part of the product floor. The deck should be impressive at first glance and durable under close review. If preserving visual quality requires a less editable export, visual quality wins. A visually correct PPTX is more important than a mechanically editable but degraded one.

## What This Is Not

This project is not trying to make the fastest possible PPT.

It is trying to make the right PPT:

- not a bullet dump
- not a decorated article
- not a generic business template
- not a website disguised as slides
- not an HTML demo with a PPTX afterthought
- not a deck that passes validation while failing human judgment

The core idea is simple: agents should not merely generate slides. They should think like presentation editors, design directors, and quality reviewers, then produce a visually ambitious PowerPoint deck worthy of that standard.
